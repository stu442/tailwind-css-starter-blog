# Series Feature Test Cases

## 1. 기본 검증

### TC-01 비시리즈 포스트 상세
- 조건: `series`가 없는 포스트
- 확인:
  - 상세 페이지 정상 렌더링
  - 기존 전역 이전/다음 링크 동작
- 기대 결과: 기존 동작과 동일

### TC-02 시리즈 포스트 상세
- 조건: `series`, `seriesOrder`가 있는 포스트
- 확인:
  - 시리즈 링크 노출
  - 이전/다음이 시리즈 내부 기준으로 동작
- 기대 결과: 시리즈 흐름으로 탐색 가능

## 2. 시리즈 페이지

### TC-03 시리즈 인덱스 `/series`
- 확인:
  - 시리즈 목록 노출
  - 항목 클릭 시 `/series/[slug]` 이동
- 기대 결과: 전체 시리즈 탐색 가능

### TC-04 시리즈 상세 `/series/[slug]`
- 확인:
  - 해당 시리즈 포스트만 표시
  - 정렬이 `seriesOrder` 우선, `date` 보조로 적용
- 기대 결과: 의도한 연재 순서 유지

### TC-05 잘못된 시리즈 slug
- 조건: 존재하지 않는 slug 접근
- 확인: 404 처리
- 기대 결과: 예외 경로 정상 처리

## 3. 엣지 케이스

### TC-06 `seriesOrder` 누락
- 조건: 일부 포스트에 `seriesOrder` 없음
- 확인: 누락 글이 후순위로 배치되고 날짜 정렬 유지
- 기대 결과: 정렬 안정성 보장

### TC-07 `seriesOrder` 중복
- 조건: 동일한 `seriesOrder` 값 존재
- 확인: 날짜 보조 정렬 적용
- 기대 결과: 렌더링 순서가 일관적

### TC-08 시리즈명 표기 차이
- 조건: 시리즈명 공백/대소문자 차이 데이터
- 확인: slug 기준으로 그룹핑 일관성 확인
- 기대 결과: 의도치 않은 분리 최소화

## 4. 빌드/품질 게이트

### TC-09 정적 검사
- 명령: `yarn lint`
- 기대 결과: 오류 없음

### TC-10 프로덕션 빌드
- 명령: `yarn build`
- 기대 결과: 빌드 성공, 라우트 생성 정상

## 5. 회귀 확인

### TC-11 기존 핵심 경로
- 경로: `/`, `/blog/[slug]`, `/tags`
- 기대 결과: 시리즈 기능 도입 전과 동일하게 동작

### TC-12 sitemap/metadata
- 확인: 시리즈 라우트가 sitemap/metadata에 반영됨
- 기대 결과: SEO 노출 대상 누락 없음

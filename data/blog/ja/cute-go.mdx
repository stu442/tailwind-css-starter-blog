## ⚙️ Golangの特徴

Goは簡潔で速い言語です。
実用的で、明確で、単純です。

### 🍲 簡潔で明示的な、まるで鶏の水炊きのような味わい

```go
func main() {
  foo := "Hello world"
  fmt.Println(foo)
}
```

コンソールに「Hello world」を表示するためのコードです。

プログラミング言語を学んだことがある方なら、本当にすぐに学べます。
変数宣言、関数、if、for、switch文、すべてが簡潔です。

### 🚀 実用主義

Goはコンパイル言語です。
しかし、コンパイル速度が非常に速いです。

さらに、すべての依存関係を含めて
一つの実行ファイルにコンパイルされるため、
デプロイも簡単です。

本当に「開発者に優しい」言語だと思います。

### 🧵 同時実行性

Goは並列プログラミングを容易にするように設計された言語です。
非同期ロジックを簡単に扱うことができます。

```go
package main

import (
	"fmt",
	"time"
)

// 何かをする関数（例：API呼び出し、ファイル処理など）
func doSomething(name string, done chan string) {
	time.Sleep(2 * time.Second)
	done <- fmt.Sprintf("%s の作業完了！", name)
}

func main() {
	done := make(chan string)

	go doSomething("A", done) // ゴルーチンとして実行
	go doSomething("B", done) // もう一つのゴルーチンを実行

	// 結果を受け取る
	fmt.Println(<-done)
	fmt.Println(<-done)
}
```

`go`というキーワードと共に関数を呼び出します。

そしてチャネルを作り、ゴルーチンが結果をやり取りできるようにすれば、
簡単かつ明確に同時に実行できるコードを作ることができます。

## 🧠 Goという言語が思考方法に与えた影響

### 📌 ポインタ概念の明確化

Goを学ぶ前は、実はポインタという概念をよく理解していませんでした。
ただ「ああ、こういう概念があるんだな」程度に漠然と知っていました。

JavaScriptでは、変数の参照とコピーを経験的に体得してきましたが、
その下のメカニズムはぼんやりとしか感じられませんでした。

しかし、Goでは「ポインタ」という概念が明示的に登場します。
最初は複雑で難しいと思いましたが、
知れば知るほど「ああ、だからポインタがあるんだな」と思うようになりました。

おかげで、ポインタに関する概念をしっかりと固めることができ、
JavaScriptのような言語でも、参照とコピーを低レベルで理解できるようになりました。

### 🧯 エラーは隠さず、直面せよ

Goで印象的だったのは、エラー処理の方法です。
ほとんどの言語には`try-catch`があります。

まとめてエラーを処理する方法です。

実は、私はこの方法があまり好きではありませんでした。
あまりにも広い範囲に`try-catch`を使うと、
正確にどの部分でエラーが発生したのか分かりにくかったからです。

Goはエラーを戻り値として明示的に作ります。

```go
package main

import (
	"errors",
	"fmt"
)

// stringとerrorを返すgreet関数
func greet(name string) (string, error) {
	if name == "" {
		return "", errors.New("名前がありません")
	}
	return "こんにちは、" + name, nil
}

func main() {
	msg, err := greet("")
	if err != nil {
		fmt.Println("エラー:", err)
		return
	}
	fmt.Println(msg)
}
```

最初は面倒かもしれませんが、
これがどれほど明確で安全な方法か分かるでしょう。

エラーをコントロールできるという自信を持つようになりました。

エラーを隠さず、明確に表に出すこと。
プログラミングだけでなく、私の人生においても有効な私だけの哲学となりました。

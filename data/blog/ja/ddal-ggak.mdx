初めて開発を勉強するようになったきっかけは、

大学生の時にお小遣い稼ぎでやっていた委託販売のプロセスを自動化したかったからです。
単純な繰り返し作業を、毎日やりたくはなかったのです。

> 「自分が必要なツールを、自分が一番使いやすい形で作る」

その考えは今も変わりません。

## 新社会人は大変です…。

新社会人として、毎月60万ウォンが出ていく生活は
非常に厳しい状況です。

そんな皆さんのために、国は「青年」のための様々な対策を用意しています。

まず、ほとんどの青年がご存知の「青年月額家賃支援」があります。
最大20万ウォンまで1年間、生涯1回支援する政策です。

私は今回申請した状態です。ふふ…

そして、意外と知られていない「青年公共賃貸」
という制度もあります。

簡単に言うと、安く住居費を解決できる、
国が安い賃貸住宅を提供する制度だと考えればいいでしょう。

## しかし…

しかし、問題があります。

まず、競争が半端ではありません。

大韓民国の多くの青年がソウルで働いているだけに、
多くの人々がソウルで安定した住居を探しています。

しかし、これは仕方ありません。

最大の問題は、「申請」が難しいということです。
毎週、ホームページにアクセスして新しい公告が出たか確認しなければなりません。

これがとても面倒でした。

## 確認「してくれ」

そこで、これも自動化すればいいと考えました。

Github Actionを利用して、
毎日、希望のウェブページに新しい記事が作成されたか確認し、
作成されていれば、テレグラムで知らせてくれるシステムです。

## 技術選定

言語から考えてみました。
ふと、以前Goを使ってクローリングをした記憶が蘇りました。

Goという言語にそれなりの楽しさを感じていたし、
勉強も兼ねてGoを選びました。

しかし、Claude Codeを使って
ほとんどの作業をする予定だったので、実はあまり意味はありませんでした…

Github ActionのCron Jobを使って、
毎日9時にテレグラムのBot APIで私の携帯電話に通知を送るように
作りたいと思いました。

## Claudeの剣舞

ある程度企画がある状態だったので、Claudeに大まかに頼みました。

すると、一人ですらすらと、30分ほどで
すべてのコードを作り上げました。

Github Actionの設定やテレグラムボットの設定など、
コード以外の部分で私がやるべきことが多かったですが、

Claude Codeの完璧なコードのおかげで
1時間で目標のシステムを完成させました。

![](/static/images/ddal-ggak/1.png)

## 自分だけのツールを作るのが一番簡単な時代

今や、本当に自分に必要で、自分に合ったツールを一人で作ることは
もはや大したことではなくなりました…

委託販売のプロセスを自動化するためにうんうん唸っていた昔の私は
こうして開発者になりましたが、

今ではツールを作るために大きな悩みなく「プロンプト」で命令を下せばいいのです。

本当に便利な世の中ですが、
明らかに失ったものも多いと思います。

問題と直接ぶつかり、転がりながら得られた、
その深く、固い成長の機会をです。

開発者として多くのことを考えます。

勉強でもしに行きます。
読んでくださってありがとうございます。
